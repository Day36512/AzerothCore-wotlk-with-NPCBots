AzerothCore / TrinityCore SQL fragment

This exporter uses the most recent forward result. It writes only the columns you are most likely to want while hand-building an item: item level, stats, sockets, armor, damage, and delay. It deliberately does not touch spell procs, required classes, bonding, source, display ID, or name. That is a feature, not cowardice.

Reference model behavior

When loaded, weapon damage comes from nearby real items filtered by quality, inventory slot, subclass, and item level. Armor comes from nearby real armor of the same slot and armor type. Passive stats use the original budget math as a scaffold, then scale the budget by what nearby real items actually spend. Robust estimates trim obvious outliers before they affect the final value.

Model notes

This build keeps the original calculator’s strongest idea: Wrath itemization is mostly a budget envelope using the exponent log(2)/log(1.5), quality curves, slot coefficients, and stat cost modifiers. The important v4 change is that the calculator can now ingest your actual 3.3.5 item_template export and use real nearby items as reference points before falling back to formulas.

What changed in this build

  • Bidirectional workflow: item level to stats, and stats back to likely item level.
  • Separate forward and reverse paths for passive stats, sockets, armor, and weapon damage.
  • v4 reference mode: load a full item_template CSV/JSON export and the forward builder uses nearest-neighbor interpolation for weapon DPS, damage spread, armor, and passive budget scale.
  • v3 correction retained: pre-200 one-hand weapons use observed TBC/Wrath DPS anchors, a lower passive-stat weapon slot coefficient, and TBC narrow-dagger spread for Malchazeen/Emerald Ripper style weapons when no DB is loaded.
  • Dynamic stat modifiers for stamina, spell power, MP5, block value, bonus armor, sockets, and patch-sensitive oddities.
  • Confidence bands and warnings for trinkets, relics, proc-heavy items, late-tier blanket ilvls, and physically weird weapon/armor fits.
  • SQL export for quick AzerothCore item_template patching.

Important honesty clause

This is now a reference-data build. Without an imported database, it behaves as a calibrated hybrid formula model. With an imported database, it becomes a local corpus-guided generator: it looks at actual nearby items in your export and uses those as the first authority. That means your custom database, not my increasingly embarrassed hard-coded guesses, gets the final vote.

Outlier policy

When the calculator returns an item-level range instead of one exact answer, that is intentional. Item level in Wrath is a budget envelope. ICC 277, Ruby Sanctum, proc trinkets, random-suffix gear, PvP resilience pieces, and handcrafted rewards regularly deviate from tidy formula math. The tool flags those cases instead of hiding the uncertainty.
